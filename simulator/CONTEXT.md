## 개편 이력

## 파일 구조
- `index.html` — 2개 탭: ①수당 계산 ②성장 시뮬레이션
- `engine.js` — 계산 엔진 (브라우저 `window.WellanceEngine` / Node `require`)
- `verify.js` — 검증 테스트. 실행: `node simulator/verify.js`
- `legacy-simulator.html` — 구 4업종 시뮬레이터 (Phase 2 재확장 참고용)

## 화면 구성
- **탭 A · 수당 계산** — 패스트팩/정기구독 인원 → 그 시점 직급·5수당 단발 계산
- **탭 B · 성장 시뮬레이션** — 1~3년 성장 추이 (`simulateGrowth`)
  - 유형: 사업자 추천(팀원 1명씩 복제 → 삼각수) / 정기구독 판매(선형 증가)
  - 주기: 매주 / 매월 (매월 = 4주 주기 추천)
  - 기간: 1년 / 2년 / 3년
  - 출력: 누적 수입, 직급 도달 시점, 월별 요약, 누적 수입 바차트, 주별 상세

## 계산 모델 (engine.js)
입력: `{ fastPackRecruits, subscriptionRecruits }`

- 본인 팩 포함(N+1팩)을 좌·우 균형 배치 → 소실적(작은 라인) 자동 환산
  - 소실적 패스트팩 = floor((N+1)/2)팩, 1팩 = 825,000 QV / 412,500 CV
  - 정기구독 1명 = 58,800 QV/CV (건기식 메타부스트 제니스, 12개월 약정가 기준)
  - 리더 4주 유지금액 45,000이 다운라인으로 이월 → 소실적 QV/CV에서 차감
- 직급: 소실적 QV로 자동 판정 (3명→브론즈, 7명→실버, 15명→골드, 29명→플래티넘)
  - 소실적 QV는 교육자료 STEP1~4의 우측(소실적) 값과 일치
  - 브론즈 미달이어도 활동(판매) 시 → **웰런스 파트너** 등급, 팀수당 10% 지급
  - 좌·우 라인 미충족(추천 2명 미만) → 자격 미달, ① 팩추천만 발생
- ① 팩추천·우대고객 커미션 = 패스트팩 인원 × 247,500원 + 정기구독 인원 × 14,994원 (직급 무관)
- ② 후원수당(팀커미션) = 소실적 CV × 직급별 지급율 (주극점 상한)
- ③ 추천매칭 = 팀커미션(주극점 적용 전) × 직급별 매칭율 — 고직급에서 ②가 막혀도 ③는 조직 규모대로 증가
- ④ 랭크업 = 직급 최초 달성 1회
- ⑤ 직급유지 = 직급별 주급액 (직전 4주 기준 유지 시 지급 — 안내문구)

## v3 모델 ↔ 요청서 TC expected 차이 (해석 차)
요청서 v1.0의 TC-1~4 expected는 "도달 첫 주 증분(incremental)" 모델 기준:
- ③ 매칭 = 자격유지 차감 **전** smallCV × teamRate × matchRate
- ⑤ 직급유지 = 도달 첫 주는 **0** (4주 후부터 지급)

현재 v3는 "총 N명 정착 시 매주 받는 수당(steady-state)" 모델:
- ③ 매칭 = 자격유지 차감 **후** smallCV × teamRate × matchRate
- ⑤ 직급유지 = 직급 유지 가정하에 매주 지급

⇒ verify.js [H] 섹션은 **v3 모델 기준 expected** 값으로 PASS. 예비 사업자 모집 도구로서 "이 단계 유지하면 매주 이만큼" 메시지가 더 직관적·설득력 있음.
